## Session 2

### Project
app-redesign

### Goal
Implement Fitbit Web API integration to replace Google Fit (which shut down June 2025)

### Deliverables
- `app/src/lib/fitbit.ts` — Core Fitbit module (PKCE, OAuth, token management, weight sync, connect/disconnect)
- `app/src/app/api/sync/fitbit/route.ts` — OAuth initiation with PKCE + CSRF state cookies
- `app/src/app/api/sync/fitbit/callback/route.ts` — OAuth callback with state validation, token exchange, initial sync
- `app/src/app/api/sync/fitbit/pull/route.ts` — Manual sync endpoint with 5-min rate limit
- `app/src/app/api/sync/fitbit/disconnect/route.ts` — Disconnect (clears tokens, keeps data)
- `app/src/app/api/sync/fitbit/mock-connect/route.ts` — Dev-only mock that inserts 30 days of sample weight data

### Decisions
- Replaced Google Fit with Fitbit as the weight sync source — Google Fit REST API shut down June 2025, Fitbit (owned by Google) still has a live API
- Used kg-to-lbs heuristic (values < 100 treated as kg) since Fitbit returns weight in user's unit preference
- Added PKCE (required by Fitbit) via code_verifier cookie alongside existing CSRF state cookie pattern
- Fixed hardcoded "Dexcom" reconnection toast to use dynamic source name
- Mock route generates 30 days of weight data with realistic GLP-1 weight loss trend (~6 lbs over 30 days)
