### Decisions
- Used AES-256-GCM for token encryption with hex-encoded 32-byte key from ENCRYPTION_KEY env var
- Dexcom OAuth state stored in httpOnly cookie (not session) for CSRF protection
- Initial sync fires non-blocking after OAuth callback — user sees profile immediately while sync runs in background
- 5-minute rate limit on manual sync to prevent button mashing
- Disconnect preserves all existing glucose data — only removes tokens
- Created mock-connect route because Dexcom sandbox SSL is broken (dexcomdev.com cert issue)
- Mock route generates realistic glucose curve with meal spikes (breakfast/lunch/dinner patterns)
